# Solutions Hub

## What This Is

A deployment platform where each git branch automatically becomes a live website. Push code to a branch, get a unique URL. Protected behind Cloudflare Access (email PIN to michal.bravansky1@gmail.com).

- **Repo:** github.com/MichalBravansky/solutions-hub
- **Production:** https://solutions-hub-1rh.pages.dev/
- **Branch previews:** `<branch-name>.solutions-hub-1rh.pages.dev`

## How Deployment Works

Everything in `public/` is deployed as a static site. There is no build step, no framework, no bundler. Just static HTML/CSS/JS.

- Push to `main` → deploys to production
- Push to any other branch → deploys to a preview URL
- GitHub Actions (`.github/workflows/deploy.yml`) handles all deployments via Cloudflare Pages
- On main deploys, the action fetches active deployments from the Cloudflare API and writes `public/deployments.json` — this powers the deployment listing on the production page

## Your Workflow as a Claude Code Session

You are expected to create solutions and deploy them. Follow this process:

1. **Create a new branch** from `main`:
   ```
   git checkout -b <descriptive-branch-name>
   ```

2. **Do your work** inside `public/`. All web content goes there. You can add new HTML files, subdirectories, CSS, JS — anything that runs in a browser.

3. **Commit and push the branch:**
   ```
   git add public/
   git commit -m "Description of what you built"
   git push -u origin <branch-name>
   ```

4. The push triggers a GitHub Action that deploys your branch. Your work will be live at `<branch-name>.solutions-hub-1rh.pages.dev` within ~30 seconds.

5. **Do NOT push to `main`** unless explicitly told to. Main is production.

## Rules

- **All web content goes in `public/`.** Do not put files outside of it unless they are config files (like this one).
- **Do not modify `.github/workflows/`** unless asked. The deploy pipeline is already configured.
- **Do not modify the production `public/index.html`** on main. It is the deployment dashboard. Build your solutions in separate files or subdirectories within `public/`.
- **Do not commit `.env`** — it contains Cloudflare credentials and is in `.gitignore`.
- **No build tools required.** Write static HTML, CSS, and JS directly. No npm, no bundlers.
- **Branch naming:** use descriptive kebab-case names like `feat-calculator`, `fix-landing-copy`, `experiment-3d-viz`. The branch name becomes the subdomain.
- **Keep branches focused.** One solution per branch.
